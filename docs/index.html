Paths

Create a new SME user

POST /v1/auth/create-user

Create any type of SME user

application/json

Request body JSON pass to API

{
    "email": "bob@newpath.com,",
    "first_name": "Bob",
    "job_title": "developer,",
    "password": "@Abc123",
    "site_id": "01G562W6T8HCE2ZDCANTPTFV1Q",
    "surname": "Alic,"
}
first_name: string

User first name

surname: string

User surname

job_title: string

User job title

email: string

User email. Only organisation emails are allowed when SME admin register first time

password: string

User password

site_id: string

Site ID belong this user

200 OK

Returns a created user information.

Example for application/json
{
    "data": {
        "email": "bob@newpath.com",
        "first_name": "Bob",
        "job_title": "developer",
        "site_id": "01G562W6T8HCE2ZDCANTPTFV1Q",
        "surname": "Alice"
    },
    "messageCode": "UserCreatedSuccessfully"
}
messageCode: string

API response status message code

first_name: string

The created user first name

surname: string

The created user surname

job_title: string

The created user job title

email: string

The created user email

site_id: string

The created user site ID

400 Bad Request

Error field validations

Example for application/json
{
    "message": "UserAlreadyExists",
    "statusCode": 400
}
message: string

API response status message code. Possible codes ['UserAlreadyExists', 'OnlyOrgEmailsAreAllowed']

statusCode: string

Http error code

Send forgot password request

}

    },

}
message: string

API response status message

email: string

Requested email

site_id: string

Requested site ID

400 Bad Request

Example for application/json

    "statusCode": 400
}
message: string

API response status message code. Possible codes ['InvalidEmailAddress', 'EmailNotVerified']

statusCode: string

Http error code

Verify SME user email address

GET /v1/auth/verify-account

When SME register with the system, email address has to be verify

application/json

token

Verification token

 query object
200 OK

Token verified successfully.

Example for application/json
{
    "data": {
        "token": "xyz..."
    },
    "message": "EmailVerifiedSuccessfully"
}
message: string

API response status message

Verified token

400 Bad Request

Token expired.

Example for application/json
{
    "message": "TokenExpired",
    "statusCode": 400
}
message: string

API response status message code. Possible codes ['TokenExpired', 'TokenInvalid', 'EmailAlreadyVerified']

statusCode: string

Http error code

Returns status of list of AWS services used by Techpath app

POST /v1/common/status

Some response properties can decide health of the application

application/json

Request body JSON pass to API

{
    "time": "2022-04-19 07:22:17"
}
time: date

Current browser time in unix format

200 OK

Returns a bunch of information.

Example for application/json
{
    "data": {
        "client_time": "2022-04-19 07:22:17",
        "server_time": "2022-04-25T07:29:21.779Z",
        "services_status": {
            "dynamodb": "failed"
        }
    },
    "message": "success"
}
message: string

API response status message

client_time: string

The time passed from browser

server_time: string

The server time

dynamodb: string

The status of dynamoDB connection from application

Get user profile

userId path object
200 OK

Returns user profile.

    "data": {
        "requestId": "01G741FCNTEENK8ZB3YKB6HJYG",

                    "SMEDeliveryName": "dddd"
                },

                    "deliveryAddressLine1": "sss",
                    "deliveryAddressLine2": ""
                }
            },

            "approver": null,
            "approvingCount": 2,
            "budgetUsed": 3000,

            "role": "admin",
            "tags": [
                "Everyone"
            ],
            "userId": "01G741FCNTEENK8ZB3YKB6HJYG"
